Serverless Image Upload & View Application

Overview

This project demonstrates an image upload and viewing system using a serverless architecture. It features a frontend React application, Node.js backend services, and AWS resources like S3, Lambda, and SQS for efficient file management.

Key Features

  • Pre-signed URL Image Upload: Images are uploaded directly to AWS S3 via pre-signed URLs generated by the backend service.
  • Serverless Architecture: AWS Lambda processes image uploads and stores file metadata in a relational database.
  • Asynchronous Processing: AWS SQS is used to queue image processing tasks and ensure scalable performance.
  • Image Viewing: A React frontend retrieves image data from the database, allowing users to view uploaded images.

Components

Frontend (React with Typescript)

  • A user-friendly interface for uploading images.
  • Fetches the pre-signed URL from the Node.js service to enable direct upload to S3.

Backend (Node.js)

  • Generates pre-signed URLs for secure image uploads.
  • Sends metadata to AWS SQS for processing by AWS Lambda.

AWS Lambda

  • Processes SQS messages and updates the MySQL database with image details.
  • Fully serverless, scales automatically with demand.

Database (AWS RDS)

  • A relational database that stores image metadata (e.g., image URL, upload timestamp).

Architecture

  • Frontend: React (Typescript, Material UI)
  • Backend: Node.js (Express) for API and AWS pre-signed URL generation
  • Database: AWS RDS (MySQL)
  • Image Storage: AWS S3 for scalable storage
  • Processing: AWS Lambda and SQS for handling image upload events and saving metadata

Installation

1. Clone the repository:

git clone https://github.com/deepikasandp/ImageUpload.git

2. Install dependencies for the frontend and backend:

- Frontend:

cd frontend
npm install
npm start

- Backend:

cd backend
npm install
npm start

3. Deploy the Lambda function and configure AWS resources (S3, SQS, RDS).

How to Use

  1. Upload an image using the frontend interface.
  2. The image is uploaded to S3 via a pre-signed URL.
  3. SQS queues the image metadata for Lambda processing.
  4. AWS Lambda stores the image metadata in the RDS database.
  5. The image can be viewed via a token number on the image viewer page.

Technologies Used

  • React (Typescript, Material UI)
  • Node.js (Express)
  • AWS S3, Lambda, SQS, RDS (MySQL)
  • Serverless architecture for scalability
